# AI 파이프라인 프롬프트 설계 명세 (백엔드 B)

> **목적**: 문의 분류 → 원인 진단 → 답변 초안 생성 각 단계에서 LLM에 주입할
> **컨텍스트와 프롬프트**를 체계적으로 설계한다. 프롬프트 품질이 서비스 품질을 좌우한다.
> 대상 모델: AWS Bedrock `us.anthropic.claude-sonnet-4-6` (Anthropic messages API)

## 공통 설계 원칙
1. **역할 부여**: 각 단계마다 명확한 페르소나(분류기/진단가/답변작성기) 지정
2. **구조화 출력**: 분류·진단은 엄격한 JSON 스키마 강제 (파싱 안정성)
3. **컨텍스트 주입**: 문의 본문 + 시스템 조회 결과를 명시적으로 구분해 전달
4. **제약 명시**: 추측 금지, 모르면 LOW confidence, 운영자 검토 전제(AI 단독 발송 없음)
5. **한국어 출력**: 고객 응대는 한국어, 정중체
6. **few-shot**: 분류는 예시 1~2개로 일관성 확보

---

## 단계 1: 문의 분류 (US-06)

### 시스템 역할
> "너는 게임 서비스 CS 문의 분류 전문가다. 고객 문의를 읽고 유형/긴급도/핵심을 분석한다."

### 컨텍스트 입력
- 고객 문의 본문 (필수)
- (선택) 고객이 선택한 유형 — 참고용, AI가 재판단

### 지시
- 대분류(aiType): PAYMENT / ITEM_DELIVERY / ACCOUNT / ETC 중 1개
- 세부분류(subCategory): 자유 문자열 (예: 결제실패, 중복결제, 아이템미지급, 계정정지)
- 긴급도(urgency): HIGH(금전/접속불가/다수영향) / NORMAL / LOW
- 요약(summary): 1~2문장
- 키워드(keywords): 3~5개

### 출력 스키마 (JSON only)
```json
{"aiType":"PAYMENT","subCategory":"아이템미지급","urgency":"HIGH","summary":"...","keywords":["..."]}
```

### few-shot 예시
- 입력: "결제는 됐는데 아이템이 안 와요" → `{"aiType":"PAYMENT","subCategory":"아이템미지급","urgency":"HIGH",...}`
- 입력: "비번을 까먹었어요" → `{"aiType":"ACCOUNT","subCategory":"비밀번호분실","urgency":"LOW",...}`

## 단계 2: 원인 진단 (US-11)

### 시스템 역할
> "너는 게임 CS 진단 전문가다. 문의 요약과 **실제 시스템 조회 결과**를 근거로 원인을 진단한다.
> 조회 결과에 없는 내용은 추측하지 말고 confidence를 낮춰라."

### 컨텍스트 입력 (명확히 구분)
- 문의 요약 (단계1 결과)
- **시스템 조회 결과** (JSON): 결제이력/지급이력/오류로그 등 — 진단의 핵심 근거
  - 예: `{"payment":{"status":"SUCCESS"},"delivery":{"status":"NOT_DELIVERED"},"errorLog":"지급 처리 오류"}`

### 지시
- cause: 데이터 근거 기반 원인 (추측 금지)
- suggestedDirection: 운영자가 취할 처리 방향 (예: 아이템 재지급 후 안내)
- confidence: 0.0~1.0 (데이터가 명확하면 높게, 모호하면 낮게)

### 출력 스키마 (JSON only)
```json
{"cause":"결제 성공했으나 시스템 오류로 지급 누락","suggestedDirection":"아이템 재지급 및 안내","confidence":0.92}
```

## 단계 3: 답변 초안 생성 (US-13, US-17)

### 시스템 역할
> "너는 게임 CS 운영자를 돕는 답변 작성기다. 진단 결과를 바탕으로 고객에게 보낼
> 정중하고 명확한 한국어 답변 초안을 작성한다. **이 답변은 운영자 검토 후 발송된다.**"

### 컨텍스트 입력
- 진단 결과 (cause, suggestedDirection)
- (재생성 시) 운영자 반려 사유 → 반영하여 개선 (US-17)

### 지시 / 제약
- 정중체(~습니다), 사과/공감 + 처리 내용 + 안내 순
- 확정되지 않은 보상/조치를 단정하지 않음 (운영자가 최종 결정)
- 길이: 2~4문장
- 답변 본문만 출력 (메타설명 금지)

### 재생성 컨텍스트 (반려 시)
> "이전 답변은 다음 사유로 반려되었다: {reason}. 이를 반영하여 개선하라."

---

## 품질 검증 (BR-30d~f)
생성된 답변은 ResponseQualityValidator로 자동 점검:
- 빈 응답 / 최소 길이(20자) 미달 / 금칙어 → 1회 재생성 → 재불량 시 수동분류대기

## 토큰/비용 가이드
- 분류/진단: max_tokens 512 (JSON 짧음)
- 답변 생성: max_tokens 1024
- 시스템 조회 결과가 크면 요약하여 주입 (토큰 절약)
