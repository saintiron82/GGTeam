## 1. 사용자 그룹 & 진입점

| 사용자 | 진입점 | 인증 | 화면 |
|--------|--------|------|------|
| 고객 | 공개 URL | 불필요 | 문의 접수 폼 |
| 운영자 | 관리자 URL | JWT 필요 | 로그인 → 칸반 보드 → 상세 → 편집기 |

## 2. 전체 화면 목록

| ID | 화면 | 경로(권장) | 담당 스토리 | 인증 |
|----|------|-----------|------------|------|
| S1 | 고객 문의 접수 폼 | `/inquiry` | US-03~05 | 불필요 |
| S2 | 문의 접수 완료 | `/inquiry/complete` | US-03, US-04 | 불필요 |
| S3 | 운영자 로그인 | `/login` | US-01, US-02 | 불필요 |
| S4 | 칸반 보드 (메인) | `/board` | US-12, US-20, US-22 | 필요 |
| S5 | 문의 상세 | `/inquiries/:id` | US-15, US-18, US-21, US-23 | 필요 |
| S6 | 답변 편집기 | S5 내 영역/탭 | US-14, US-16, US-17, US-24 | 필요 |
| S7 | 목록/리스트 뷰 | `/inquiries` | US-22 | 필요 |

> S6(편집기)는 S5(상세)와 같은 페이지 안에서 동작하는 영역으로 구현 권장. 별도 라우트 아님.
> S4(칸반 보드)와 S7(리스트 뷰)은 동일 데이터를 보는 두 가지 뷰. 운영자가 보드/리스트 토글로 전환한다.

## 3. 네비게이션 흐름

```
[고객]
  S1 문의 폼 ──제출──> S2 접수 완료(문의 ID 표시)

[운영자]
  S3 로그인 ──성공──> S4 칸반 보드
                         │
                         ├─ 카드 클릭 ─────────> S5 문의 상세
                         │                          │
                         │                          ├─ 담당하기(Pull) → 상태: 운영자확인중
                         │                          ├─ S6 답변 편집/승인/반려/재분석
                         │                          └─ 타임라인/이력 확인
                         │
                         ├─ 필터/검색 ───────────> S4(필터 적용)
                         ├─ 보드/리스트 토글 ──────> S7 리스트 뷰 (같은 데이터, 표 형태)
                         └─ 알림 배지(미배정/긴급) → 해당 카드 강조
  토큰 만료(401) ──────> S3 로그인 리다이렉트

[S7 리스트 뷰] ──행 클릭──> S5 문의 상세 (S4와 동일 진입)
```

## 4. 전역 레이아웃 규칙

- **운영자 영역**: 좌측 또는 상단 네비 + 본문. 상단에 알림 배지(미배정/긴급 카운트) 상시 노출.
- **고객 영역**: 네비 없는 단일 폼 중심 레이아웃 (브랜딩 최소).
- **인증 가드**: 운영자 화면(S4~S7)은 ProtectedRoute로 감싸고, 미인증 시 S3로 리다이렉트.

## 5. 화면 우선순위 (구현 순서 권장)

1. S3 로그인 → S4 칸반 보드 (운영자 메인 동선)
2. S5 문의 상세 + S6 편집기 (핵심 처리 동선)
3. S7 리스트 뷰 (칸반과 토글, 필터/검색/페이징 중심)
4. S1/S2 고객 문의 폼 (독립 동선)
