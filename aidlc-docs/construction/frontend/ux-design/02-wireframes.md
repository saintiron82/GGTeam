# 02. 와이어프레임 명세

> 시각 프로토타입은 `wireframe/`(포트 5174)에서 실제 렌더링으로 확인.
> 본 문서는 각 화면의 레이아웃·구성요소·표시 규칙을 텍스트로 고정한 명세다.
> 대응 코드: `wireframe/src/screens/*.tsx`

---

## S3. 운영자 로그인 (`LoginScreen.tsx`)

```
+------------------------------------------+
|        AI CS 문의 처리 에이전트            |
|                                          |
|   아이디    [____________________]        |
|   비밀번호  [____________________]        |
|                                          |
|            [   로그인   ]                 |
|                                          |
|   * 실패: "아이디 또는 비밀번호가          |
|     올바르지 않습니다"                     |
|   * 5회 실패 시 계정 잠금                  |
+------------------------------------------+
```
- 중앙 정렬 카드(폭 ~360px)
- 필드: username, password
- 에러 메시지 영역 (INVALID_CREDENTIALS / ACCOUNT_LOCKED 대응)

## S1. 고객 문의 접수 폼 (`InquiryFormScreen.tsx`)

```
+--------------------------------------------------+
|  문의하기                                          |
|  유저 ID   [____________]                         |
|  닉네임    [____________]                         |
|  문의 유형 (필수) [▼ 결제/아이템지급/계정/기타]     |
|  문의 내용 (필수, 최소 10자)                       |
|  [                                          ]    |
|  [                                          ]    |
|            [  문의 제출  ]                        |
|  * 제출 성공 → 문의 ID + "접수 완료"               |
+--------------------------------------------------+
```
- 유형 미선택 / 내용 10자 미만 시 제출 불가 (VALIDATION_ERROR 대응)
- 제출 성공 시 S2(완료)로 전환, 문의 ID 노출

## S4. 칸반 보드 (`KanbanScreen.tsx`)

```
+--------------------------------------------------------------+
| [미배정 5건] [긴급 2건]        필터: 상태▼ 긴급도▼ 유형▼ 담당자▼ 🔍 |
+--------------------------------------------------------------+
| 접수 | AI분석중 | 담당자배정대기 | 운영자확인중 | 승인완료 | 발송완료 |
|------|---------|--------------|------------|--------|--------|
| [카드]|  [카드]  |   [카드]      |   [카드]    | [카드]  | [카드]  |
|      |         |   [카드]      |            |        |        |
+--------------------------------------------------------------+
카드: INQ-ID | 긴급도뱃지 | [유형] | 요약   (긴급=빨간 테두리)
```
- 6개 상태 컬럼 (InquiryStatus 7개 중 MANUAL_CLASSIFICATION_PENDING은 별도 처리/배지)
- 상단 알림 배지 + 보드/리스트 토글 + 필터/검색 바
- 카드 클릭 → S5 상세 이동

## S7. 목록 / 리스트 뷰 (`ListScreen.tsx`)

```
+--------------------------------------------------------------+
| [칸반|리스트]   필터: 상태▼ 긴급도▼ 유형▼ 담당자▼ 🔍 검색 📅 기간 |
+--------------------------------------------------------------+
| 문의ID | 고객유형 | AI유형 | 긴급도 | 상태 | 담당자 | 요약 | 접수시각 |
|--------|---------|-------|-------|------|-------|------|---------|
| INQ-.. | 기타     | 결제  | [긴급] | 배정대기| -    | ...  | 15:00   |
| ...                                                          |
+--------------------------------------------------------------+
|                  ◀ 1 2 3 ▶   총 53건 · 20건/페이지            |
+--------------------------------------------------------------+
```
- S4 칸반과 **동일 데이터의 표 형태 뷰**. 상단 토글로 칸반↔리스트 전환.
- 정렬·필터·검색·기간·페이징 중심 (대량 문의 탐색에 유리)
- 행 클릭 → S5 상세 이동 (칸반 카드 클릭과 동일)
- 긴급도는 뱃지로 강조

## S5. 문의 상세 (`DetailScreen.tsx`)

```
+-------------------------------------------+----------------+
| 원본 문의 (고객선택유형/유저/시각/내용)        | 처리 이력       |
+-------------------------------------------+ (타임라인)      |
| AI 분석 (ai_type/서브/긴급도/요약/키워드)      | ● 답변초안 생성  |
+-------------------------------------------+ ● 진단 완료     |
| 시스템 조회 결과 (Payment 등)               | ● 분석 완료     |
+-------------------------------------------+ ● 문의 접수     |
| AI 진단 (원인/처리방향/신뢰도)               |                |
+-------------------------------------------+ [담당하기(Pull)]|
+-------------------------------------------+----------------+
```
- 좌측 2/3: 본문·분석·조회결과·진단
- 우측 1/3: 타임라인 + 액션 버튼
- 미완료 단계 필드는 `null` → "분석 전" 등 빈 상태(empty state) 표시

## S6. 답변 편집기 (`EditorScreen.tsx`, S5 내 영역)

```
+--------------------------------------------------+
| AI 답변 초안 (편집 가능 · 재생성 N회)              |
| [ 인라인 편집 텍스트 영역                       ] |
+--------------------------------------------------+
| [승인] [수정 후 승인] [반려 및 재생성] [재분석]    |
|  · 승인 → APPROVED → 자동 발송(SENT)             |
|  · 반려 → 사유 입력(필수) → AI 재생성(최대 3회)   |
|  · 재분석 → AI_ANALYZING                         |
+--------------------------------------------------+
| 반려 사유 입력 (반려 클릭 시 노출)                |
| [                                            ]  |
| * 사유 없이 반려 불가 (REASON_REQUIRED)          |
+--------------------------------------------------+
```
- 버튼 활성 조건은 `03-ux-flows.md` 상태전이 표 기준
- 재생성 3회 초과 시 "반려 및 재생성" 비활성 (REGENERATION_LIMIT_EXCEEDED)
