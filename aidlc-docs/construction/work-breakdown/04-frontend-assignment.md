# 작업 분장: 프론트엔드 (기획 1명 + 개발 1명)

> **협업 모델**: 기획자가 화면/UX를 정의 → 개발자가 구현. API 계약(01)·공유스키마(02)를 공동 기준으로 사용.  
> 기술: React + TypeScript (Vite), Axios. (requirements.md 기준 — 일부 문서의 Next.js 표기는 Vite로 통일)

---

## 프론트 기획자 (1명)

> 화면이 무엇을, 어떻게 보여주고 동작하는지 정의. 개발자가 바로 구현할 수 있는 수준의 명세 산출.

### 산출물
1. **화면 목록 & 정보구조(IA)**
   - 전체 화면 맵, 네비게이션 흐름
2. **와이어프레임** (화면별 레이아웃)
   - 고객 문의 폼
   - 운영자 로그인
   - 칸반 보드 (상태별 6컬럼)
   - 문의 상세 (AI분석/조회결과/초안/타임라인)
   - 답변 편집기 (인라인 편집 + 승인/반려/수정후승인)
3. **UX 플로우 / 인터랙션 명세**
   - 문의 접수 → 완료 흐름
   - 운영자: Pull 배정 → 검토 → 승인/반려/재분석 흐름
   - 상태별 버튼 활성/비활성 규칙 (02 §5 상태전이 규약 기반)
   - 에러/로딩/빈 상태(empty state) 표시 규칙
4. **화면-API 매핑표** (각 화면이 호출하는 01 API)
5. **표시 규칙**: enum 한글 라벨, 긴급도 색상, 날짜 포맷(KST)

### 담당 스토리 (UX/화면 관점)
US-03~05, US-12, US-15~24 의 화면 정의

### 의존
- API 계약(01), 공유 스키마(02) — 이미 확정됨, 즉시 시작 가능

---

## 프론트 개발자 (1명)

> 기획 산출물 + API 계약 기반으로 React 구현. API 미완성 시 Mock 서버/MSW로 병렬 개발.

### 담당 모듈 (디렉토리)
- `common` — ApiClient(Axios+JWT 인터셉터), AuthContext, ProtectedRoute, 공통 컴포넌트
- `auth` — LoginPage, 토큰 관리/만료 리다이렉트
- `customer` — InquiryFormPage (문의 접수)
- `kanban` — KanbanBoardPage, 카드 컴포넌트, 필터/검색 바, 알림 배지
- `detail` — InquiryDetailPage, 타임라인 컴포넌트
- `editor` — DraftEditor (편집/승인/반려/재분석 버튼)

### 담당 스토리 (구현 관점)
US-01~05, US-12, US-14~24

### 산출물
- React 컴포넌트 + 라우팅
- API 연동 (01 계약 기반)
- 컴포넌트 테스트 (React Testing Library)
- `data-testid` 부여 (자동화 친화)

### 병렬 개발 전략
- 백엔드 API 미완성 구간은 **MSW(Mock Service Worker)** 로 01 계약 기반 모킹 → 독립 개발
- 기획 와이어프레임 완료된 화면부터 순차 구현

### 의존
- 기획자 와이어프레임/인터랙션 명세
- API 계약(01) — 확정됨. 실제 백엔드는 통합 시점에 연결
