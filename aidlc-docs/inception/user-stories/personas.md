## 페르소나 1: 고객 (Customer)

| 항목 | 내용 |
|------|------|
| **이름** | 김도윤 (28세) |
| **역할** | 게임 유저, 문의 접수자 |

### 특성
- 모바일 게임을 주로 플레이하는 직장인
- IT 기기에 익숙하지만 CS 절차에는 관심 없음
- 빠른 답변을 기대하며, 반복 문의에 불만을 느낌

### 주요 목표
- 겪고 있는 문제를 간단하게 설명하고 빠르게 해결받고 싶음
- 문의 진행 상황을 실시간으로 확인하고 싶음
- 보상이 필요한 경우 신속하게 지급받고 싶음

### 불편사항 (Pain Points)
- 문의 후 답변까지 대기 시간이 길다
- 같은 내용을 반복 설명해야 하는 상황이 발생한다
- 문의 접수 후 처리 상태를 알 수 없다
- 정형화된 답변만 돌아와서 문제가 해결되지 않는다

### 시스템 사용 시나리오
1. 게임 내 결제 오류 발생
2. 웹 폼에서 문의 유형 선택 후 상황 설명 작성
3. 접수 확인 알림 수신
4. AI가 생성한 답변을 이메일/앱 푸시로 수신
5. 미해결 시 추가 문의 진행

## 페르소나 2: 운영자 (Operator)

| 항목 | 내용 |
|------|------|
| **이름** | 박서연 (32세) |
| **역할** | CS팀원, 문의 처리 및 승인자 |

### 특성
- 3년차 CS 운영 경험, 게임 도메인 지식 보유
- 하루 평균 50~100건의 문의를 처리
- 반복 업무 효율화에 관심이 높음

### 주요 목표
- AI가 생성한 답변을 빠르게 검토하고 승인/수정하고 싶음
- 반복 문의는 자동 처리하여 업무 부담을 줄이고 싶음
- 이슈 유형별 통계를 파악하여 서비스 개선에 기여하고 싶음

### 불편사항 (Pain Points)
- 단순 반복 문의에 시간을 많이 소비한다
- 문의 우선순위 판단이 수동이라 긴급 건을 놓칠 수 있다
- 여러 시스템(게임 DB, 결제 시스템)을 오가며 확인해야 한다
- 답변 품질 일관성을 유지하기 어렵다

### 시스템 사용 시나리오
1. 대시보드에서 신규 문의 목록 확인
2. AI가 자동 분류한 우선순위 및 카테고리 확인
3. AI 생성 답변 초안 검토
4. 필요 시 답변 수정 후 승인 → 고객에게 발송
5. 자동 처리된 건은 사후 샘플링으로 품질 확인
