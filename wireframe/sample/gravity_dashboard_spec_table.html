정보 계층 재배치 — 업무 핵심 정보를 최상단으로
기존에는 프로필 → 빠른버튼 → 결재/회의 순이어서 가장 중요한 결재 대기와 오늘 회의를 보려면 눈이 아래로 내려가야 했습니다. 리디자인에서는 Row 2에 결재·회의·생일 3위젯을 가로로 배치해 페이지 로드 즉시 업무 현황이 보이도록 개선했습니다. 빠른 실행 버튼은 Row 3으로 이동 — "상황 파악 후 실행"이라는 자연스러운 시선 흐름을 만듭니다.
기존
프로필 → 빠른버튼(파란 블록 3개) → 결재 → 회의
개선
프로필+지표 → 결재+회의+생일 → 빠른실행 → 휴가+공지
빠른 실행 버튼 — 파란 블록에서 기능 카드로
기존 파란 배경 버튼 3개는 주목도는 높지만, 결재·회의 위젯과 동일한 시각적 무게를 가져 위계가 무너집니다. 리디자인에서는 흰 배경 카드에 아이콘 + 제목 + 부제 + 화살표 구조로 변경했습니다. 파란색을 아이콘 배경(EDF3FA)에만 사용해 Primary Blue의 신뢰감은 유지하되, 시각적 과부하를 줄였습니다.
기존
파란 배경 + 흰 아이콘 + 텍스트 — 3개가 동등하게 강조
개선
흰 카드 + blue 아이콘 배경 + hover 시 테두리 강조 — 세련된 기능 카드
AI-HUB 버튼 — Accent에서 Primary로 격상
기존 AI-HUB 버튼은 Accent Sky(#06ADEC)로 채워져 있어 신뢰감이 떨어지고 '기능 버튼'처럼 보였습니다. 헤더 내 유일한 CTA인 AI-HUB는 Primary Blue(#05539A)로 변경해 브랜드 신뢰감과 버튼의 중요성을 동시에 부여합니다. Accent Sky는 내부 아이콘 강조나 뱃지 한정 사용 원칙을 유지합니다.
기존
Sky Blue 배경 + 헤더 내 여러 버튼과 비슷한 무게
개선
Primary Blue + sparkle 아이콘 — 헤더 유일 강조 CTA
카드 일관성 — 모든 위젯이 동일한 카드 토큰 사용
기존에는 위젯별로 border-radius, 보더 두께, 패딩이 제각각이어서 통일감이 없었습니다. 리디자인에서는 모든 위젯 카드에 동일한 토큰(1px border #E2E6EC / 10px radius / 16px padding / 카드 헤더 구조 통일)을 적용합니다. 카드 헤더는 항상 아이콘 + 제목(좌) + 전체보기(우) 구조로 일관성을 유지합니다.
프로필 카드 — 분산된 지표를 가로 통합 레이아웃으로
기존에는 프로필 카드(좌)와 주간근무/연차 패널(우)이 분리되어 있었고, 우측에 모바일 QR 카드가 별도로 있었습니다. 리디자인에서는 프로필 + 근무시간 + 연차를 하나의 가로 카드로 통합(8col)하고 QR을 4col 카드로 분리합니다. Divider로 세 영역을 구분해 관련 정보를 한 시선에서 스캔 가능하게 합니다.
사이드바 — 메뉴 그룹핑과 뱃지 계층 정비
기존 사이드바는 모든 메뉴가 동등한 시각적 무게를 가져 계층 없이 나열되었습니다. 리디자인에서는 메인 / 업무 / 근태·인사 / 기타 그룹 헤더로 구분하고, 처리 대기 건수는 빨간 뱃지, 안읽음은 파란 N 뱃지로 의미를 달리 표현합니다. 접기 버튼은 하단에 고정하여 레이아웃 전환을 쉽게 접근 가능하게 합니다.
